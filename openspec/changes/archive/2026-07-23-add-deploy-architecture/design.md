## Context

Ourplan is an Astro 6 static site with PWA support. Build output goes to `dist/`. Currently deploy requires manual steps (build locally, copy to server). Ourlens project already solved this with multi-stage Docker — same stack (Astro + React + Tailwind + nginx). This design ports that proven pattern.

## Goals / Non-Goals

**Goals:**
- Single `docker build` command produces a production-ready image
- Static assets served via nginx with proper caching (hashed assets = immutable, HTML = no-cache, SW = no-cache)
- Build-time env vars (`PUBLIC_*`) passed via `--build-arg`
- PWA offline page served on 404
- Minimal image size (nginx:alpine, multi-stage discards build deps)

**Non-Goals:**
- CI/CD pipeline (adds later if needed)
- Docker Compose or multi-service orchestration
- HTTPS termination (handled by reverse proxy in prod)
- Environment-specific config per stage (single Dockerfile covers all)

## Decisions

| Decision | Choice | Why |
|---|---|---|
| Base image | `node:22-alpine` for build, `nginx:alpine` for serve | Matches node engine >=22.12.0; alpine keeps images small |
| pnpm install | `--frozen-lockfile` | Fail early if lockfile out of sync — deterministic builds |
| corepack | `corepack enable && corepack prepare pnpm@<version>` | No global pnpm install needed; version pinned by `packageManager` field |
| pnpm-workspace.yaml | Copied during install step alongside package.json + pnpm-lock.yaml | ourplan has `allowBuilds` restrictions; pnpm reads this file at install time |
| Env vars | `ARG` → `ENV` in build stage | Astro substitutes `import.meta.env.PUBLIC_*` at build time |
| nginx config | Reuse existing `nginx.conf` | Already identical to ourlens — proven PWA caching strategy |
| .dockerignore | Exclude dev/test/ops dirs | Smaller build context, faster builds, no secrets leaked |
| packageManager field | Add to `package.json` | Required by corepack to auto-detect pnpm version |

## Risks / Trade-offs

| Risk | Mitigation |
|---|---|
| Missing `packageManager` = corepack falls back to system default, wrong pnpm version | Add field before Dockerfile — single source of truth |
| `.dockerignore` misses a dir → slow builds or leaked secrets | Test with `docker build .` and check context size |
| nginx 404 fallback to `/offline/index.html` assumes offline page exists | Already generated by Astro PWA plugin — verified in `dist/` |

## Context

Store audit revealed 6 correctness issues in the form store layer. The Zustand store (`src/store/form.ts`) handles 17 flat fields + 5 nested fields per support-cirlce person, persisted to localStorage. No data loss was found, but there are type inaccuracies, React anti-patterns, unnecessary subscriptions, stale persisted data, unstable React keys, and a coercive validation constraint.

All 6 fixes are self-contained — each touches 1-2 files with no cascading dependencies between them. The change is a batch of independent cleanups rather than a single coordinated feature.

## Goals / Non-Goals

**Goals:**
- Fix type inaccuracy where `useField` return type claims non-nullable values but store is `Partial<FormValues>`
- Fix React anti-pattern (side effect during render in `AuthGuard`)
- Fix unstable React key in `SummaryReviewCard` list rendering
- Remove unnecessary field subscription from `ValidatedRadioGroup`
- Stop persisting transient error state to localStorage
- Relax `hazard_flags` validation to allow empty selection

**Non-Goals:**
- No new features or external-facing behavior changes (except `hazard_flags` no longer requiring selection)
- No refactoring of the store architecture or validation system
- No changes to the session store
- No changes to Zod validation for other fields

## Decisions

| Decision | Choice | Rationale | Alternatives Considered |
|---|---|---|---|
| `useField` return type fix | Add `| undefined` to typed overload | Minimal type change; all callers already guard | Creating a separate non-nullable variant hook — overkill for 6 locations that all guard |
| `AuthGuard` redirect | `useEffect` with `[hydrated, isValid]` deps | Follows React hooks rules; no behavioral change | `useLayoutEffect` — unnecessary, flash is already hidden by loading overlay |
| `errors` persist fix | Strip from `partialize` | Errors are recomputed from field values + schemas; stale errors have no value | Keeping but expiring on mount — more complexity for no benefit |
| `ValidatedRadioGroup` cleanup | Remove `useField("parent_name")` + label replacement | Dead code — no label uses `[Name]`/`[Parent Name]` pattern | Keeping as-is — unnecessary subscription but no harm |
| `hazard_flags` fix | Remove `.min(1)` from schema | Simplest change; empty array goes from invalid to valid | Adding "None" option — more code, same effect |
| SummaryReviewCard key | `_id ?? i` | Stable unique ID already generated by `SupportCircleRepeater` | `crypto.randomUUID()` — unnecessary, `_id` already exists |

## Risks / Trade-offs

- **[Low] hazard_flags becomes optional**: Users can now submit step 3 with no hazard flags selected. This changes the data contract — downstream systems (n8n) receive empty array instead of at least one flag. Decision: product-vetted as acceptable.
- **[Low] errors removed from persist**: On page refresh, any non-submitted form errors disappear. This is correct behavior — the user sees fresh validation state after re-interaction.
- **[Low] ValidatedRadioGroup no longer supports `[Name]` replacement**: The regex pattern was dead code (0 callers). If a future component uses this pattern, it must use template literal labels instead.
